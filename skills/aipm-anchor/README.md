# AIPM-Anchor

AIPM-Anchor 是 AIPM 系列里的第一个 skill，用来帮助用户把一个模糊的 AI 项目想法、Vibe Coding demo 或已有小工具，重新锚定到一个真实业务场景里。

AIPM 的长期目标，是把“做一个适合简历和面试防守的 AI 项目”拆成一组连续 skill：从功能定义、技术拆解、评测体系、badcase 归因迭代，到最后的简历表达和面试防守。Anchor 只负责第一步：判断这个项目到底应该做什么功能，以及这个功能为什么不是一个小玩具。

## 这个 skill 解决什么

- 判断现有项目是否有真实业务场景，而不只是“AI + 行业”的概念。
- 找到场景里最值得 AI 介入的高摩擦任务节点。
- 判断功能是否具备自进化接口和主动服务接口。
- 把功能收敛成一个 3-7 天可以验证的最小 MVP。
- 给出一个更长期的系统终局方向，方便后续继续拆技术方案和评测体系。

## 交互流程

1. 诊断当前功能：用结构化维度判断它像业务功能、半成品方向，还是 AI 小玩具。
2. 锚定业务场景：从用户已有经历、项目或工具里，选择一个真实、重复、有失败成本的任务场景。
3. 定义 MVP 功能：保留可用部分，补上 AI 判断节点、反馈入口、记忆沉淀、主动触发和边界约束。

## 使用方式

在 Codex 中可以这样调用：

```text
Use $aipm-anchor to diagnose whether my AI project function is business-grounded and define a minimum MVP.
```

也可以直接描述你的项目现状，例如：

```text
我现在做了一个面试复盘 AI 工具，可以上传面试记录并生成复盘报告。请用 AIPM-Anchor 帮我判断它是不是太像小玩具，以及应该怎么改成更有业务场景的 MVP。
```
